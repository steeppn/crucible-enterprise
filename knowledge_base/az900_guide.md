# AZ-900: Microsoft Azure Fundamentals (Synthetic Certification Guide)

> SYNTHETIC DOCUMENT — Created for CRUCIBLE Enterprise hackathon demo. Not official Microsoft material.

## Exam Overview
- **Exam Code:** AZ-900
- **Level:** Fundamentals
- **Recommended Study Hours:** 15–20 hours
- **Passing Score:** 700/1000
- **Target Audience:** Beginners, non-technical roles, anyone new to cloud

## Skill Domains

### Domain 1: Cloud Concepts (15–20%)
- **Describe cloud computing benefits:** High availability, scalability, elasticity, agility, disaster recovery, and cost savings
- **Compare cloud models:** Public cloud, private cloud, hybrid cloud
- **Describe consumption-based model:** Pay-as-you-go pricing, capital expenditure (CapEx) vs operational expenditure (OpEx)
- **Key concepts to defend:** Why cloud is more cost-effective than on-premises for variable workloads; the difference between scalability (handling increased load) and elasticity (automatically scaling up and down)

### Domain 2: Azure Architecture and Services (30–35%)
- **Describe core Azure architectural components:** Regions, availability zones, region pairs
- **Describe Azure management tools:** Azure Portal, Azure CLI, Azure PowerShell, Cloud Shell, Azure Mobile App
- **Describe core Azure compute services:** Virtual Machines, Azure App Service, Azure Functions, Azure Container Instances
- **Describe core Azure networking services:** Virtual Networks, Load Balancer, Application Gateway, VPN Gateway, Azure DNS
- **Describe core Azure storage services:** Blob storage, File storage, Queue storage, Table storage, Disk storage
- **Key concepts to defend:** When to use VMs vs App Service vs Functions; the purpose of availability zones vs region pairs; difference between blob, file, and queue storage

### Domain 3: Azure Management and Governance (20–25%)
- **Describe Azure governance tools:** Azure Policy, Role-Based Access Control (RBAC), Resource Locks, Tags, Azure Resource Manager
- **Describe Azure monitoring tools:** Azure Monitor, Azure Service Health, Azure Advisor
- **Describe cost management tools:** Azure Cost Management + Billing, Azure Pricing Calculator, Total Cost of Ownership (TCO) Calculator
- **Key concepts to defend:** How RBAC differs from Azure Policy; when to use resource locks; how Azure Advisor provides recommendations

### Domain 4: Security, Privacy, Compliance, and Trust (20–25%)
- **Describe Azure security layers:** Defense in depth, network security groups, Azure Firewall, DDoS protection
- **Describe identity and access management:** Microsoft Entra ID (formerly Azure AD), Multi-Factor Authentication (MFA), Conditional Access
- **Describe compliance and privacy:** Microsoft Trust Center, Service Trust Portal, compliance offerings (ISO, SOC, GDPR)
- **Describe the Shared Responsibility Model:** What Microsoft manages vs what the customer manages
- **Key concepts to defend:** Why MFA is critical; how the shared responsibility model changes between IaaS, PaaS, and SaaS; what Azure Key Vault protects

## Common Exam Topics
- Difference between IaaS, PaaS, and SaaS with examples
- When to choose Azure Functions over Virtual Machines
- How availability zones provide fault tolerance
- The purpose of Azure Resource Manager templates
- How RBAC scopes work (management group, subscription, resource group, resource)

## Prerequisites
- None — this is an entry-level certification
- Recommended: Basic understanding of IT concepts (networking, storage, compute)

## Study Pattern Recommendation
- Study 1–2 hours daily for 2 weeks
- Take practice assessments after each domain
- Target 80%+ on practice tests before scheduling exam
